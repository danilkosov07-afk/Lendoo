# Исправление ошибки 404 на Vercel

## Основные причины ошибки 404

### 1. Неправильная корневая директория

Если ваш проект находится в подпапке `landing-studio`, а не в корне репозитория:

**Решение:**
1. Откройте проект в Vercel Dashboard
2. Перейдите в **Settings** → **General**
3. Найдите раздел **Root Directory**
4. Установите: `landing-studio`
5. Сохраните и передеплойте

### 2. Проверка структуры проекта

Убедитесь, что в корне репозитория (или в `landing-studio/`) есть:
- ✅ `package.json`
- ✅ `next.config.js`
- ✅ `tsconfig.json`
- ✅ `app/` папка
- ✅ `public/` папка

### 3. Проверка build команды

В Vercel Dashboard → Settings → General проверьте:
- **Build Command:** `npm run build` (или оставьте пустым для автоопределения)
- **Output Directory:** оставьте пустым (Next.js сам определит)
- **Install Command:** `npm install` (или оставьте пустым)

### 4. Переменные окружения

Если используете email отправку, добавьте в Vercel:
- **Settings** → **Environment Variables**
- `EMAIL_USER` = ваш email
- `EMAIL_PASS` = App Password от Gmail

### 5. Проверка логов сборки

1. В Vercel Dashboard откройте проект
2. Перейдите в **Deployments**
3. Откройте последний деплой
4. Проверьте **Build Logs** на наличие ошибок

### 6. Локальная проверка build

Перед деплоем проверьте локально:
```bash
cd landing-studio
npm run build
```

Если build проходит успешно, проблема в настройках Vercel.

## Быстрое решение

1. **Удалите проект в Vercel** (если есть)
2. **Создайте новый проект** через Vercel Dashboard
3. **При импорте репозитория:**
   - Если проект в подпапке: укажите **Root Directory** = `landing-studio`
   - Если проект в корне: оставьте пустым
4. **Деплойте заново**

## Альтернатива: Переместить проект в корень

Если хотите, чтобы проект был в корне репозитория:
1. Переместите все файлы из `landing-studio/` в корень репозитория
2. Обновите `.gitignore` если нужно
3. Закоммитьте изменения
4. Vercel автоматически пересоберет проект
